# **Product Requirements Document (PRD): Free Public LMS**

## **1\. Product Overview**

**Objective:** To build a free, high-performance, internally managed Learning Management System (LMS) for the public. The platform will deliver video, text, and auto-graded quiz content seamlessly, tracking user progress and issuing certificates upon completion. 

**Target Audience:** 

1\. **Learners (Public):** Individuals seeking free educational content with a frictionless onboarding and learning experience. 

2\. **Admins & Content Editors (Internal):** Team members who build courses, upload media, manage users, and monitor platform analytics.

## **2\. Technology Stack**

* **Backend & Core Framework:** Laravel 12.x (PHP)  
* **Frontend Framework:** React 19.x  
* **Routing & Data Bridge:** Inertia.js  
* **Starter Kit:** Laravel Breeze (React \+ Inertia stack) for scaffolding authentication.  
* **Styling:** Tailwind CSS  
* **UI Component Library:** Shadcn UI (Strict adherence to official components before custom builds).  
* **Database:** MySQL or PostgreSQL  
* **Infrastructure:** AWS S3 (or similar) for media storage, a CDN for video delivery, and Redis for caching/queues.

## **3\. Scope & Functional Requirements**

### **3.1 Authentication & Onboarding**

* **Single Sign-On (SSO):** Integration with Google and Facebook using Laravel Socialite.  
* **Standard Auth:** Email/password registration and login (provided by Laravel Breeze).  
* **Profile Management:** Ability for users to update their name, profile picture, and view earned certificates.

### **3.2 Learner Experience (Public Facing)**

* **Course Catalog:** A browsable list of all published courses.  
  * *Features:* Search bar, category filters, difficulty tags.  
  * *Action:* One-click "Start Course" (no cart/checkout).  
* **Learner Dashboard:** A personalized view of active and completed courses.  
  * *Features:* Visual progress bars, "Resume" buttons linking directly to the last viewed lesson.  
* **Course Player (The Learning Environment):**  
  * *Curriculum Sidebar:* An expandable/collapsible list of Sections and Lessons.  
  * *Content Area:* Embedded video player, rich text rendering, or PDF viewer.  
  * *Navigation:* "Previous" and "Next Lesson" buttons. Auto-marks lessons as complete when viewed.  
* **Assessments (Quizzes):**  
  * *Features:* Auto-graded multiple-choice and true/false questions. Instant feedback on pass/fail.  
* **Gamification & Completion:**  
  * *Features:* Automated PDF certificate generation upon 100% course completion and passing required quizzes.  
* **Community Support:** Static banners/buttons linking to official Facebook Groups for discussion.

### **3.3 Admin & Content Management (Internal)**

* **Role-Based Access Control (RBAC):** Super Admins (full access) and Content Editors (course management only).  
* **Course Builder:**  
  * *Features:* Create Course \-\> Add Sections \-\> Add Lessons. Ability to reorder content via drag-and-drop.  
  * *Status Management:* Draft, Review, Published.  
* **Media Management:** Uploading videos/images directly to cloud storage (S3) and linking them to lessons.  
* **Quiz Builder:** Interface to add questions, define correct options, and set passing thresholds.  
* **User Management:** Table view of all registered users with the ability to search, view progress, and suspend accounts.  
* **Analytics Dashboard:** High-level metrics showing total users, active enrollments, course completion rates, and system health.  
* **SEO Settings:** Fields to define meta titles, descriptions, and Open Graph images for individual courses and the main site.

### **3.4 Page Architecture & Route Mapping**

To ensure a clean separation of concerns and a secure application, the platform's pages are divided into three main categories based on user access levels.

#### **A. External Pages (Public / Unauthenticated)**

These pages are accessible to anyone on the internet. Their primary goal is SEO visibility, communicating the value of your free platform, and driving user registration.

* **Home / Landing Page:** \* *Purpose:* The main entry point. Features a hero section, value proposition (100% free courses), and a grid of popular or newly released courses.  
* **Course Catalog (`/courses`):** \* *Purpose:* The main browsing interface. Includes a search bar and Shadcn UI `Select` or `Toggle Group` components for filtering by category or difficulty.  
* **Course Details Page (`/courses/{slug}`):** \* *Purpose:* The public-facing syllabus. Displays the course title, description, instructor/author, and a read-only Shadcn `Accordion` showing the curriculum.  
  * *Primary Action:* A prominent "Login to Start Learning" or "Start Course" button (which triggers the auth flow if unauthenticated).  
* **Authentication Pages (`/login`, `/register`):** \* *Purpose:* Standard Laravel Breeze pages using Shadcn `Form` components, heavily featuring "Continue with Google" and "Continue with Facebook" SSO buttons.  
* **Static Pages (`/terms`, `/privacy`, `/about`):** \* *Purpose:* Necessary legal and informational pages.

#### **B. Student Pages (Authenticated Learners)**

These pages use an "App Layout" (typically featuring a persistent top navigation bar or side menu). They are strictly protected by Laravel's auth middleware.

* **Learner Dashboard (`/dashboard`):** \* *Purpose:* The learner's home base. Displays "In Progress" courses with Shadcn `Progress` bars and "Completed" courses.  
* **Course Player / Learning Environment (`/learn/{course-slug}/lesson/{lesson-id}`):** \* *Purpose:* The core consumption page.  
  * *Layout:* A specialized layout with a collapsible sidebar (using Shadcn `Sheet` on mobile and a fixed aside on desktop) containing the curriculum. The main content area renders the video player, PDF, or text.  
* **Quiz Interface (`/learn/{course-slug}/quiz/{quiz-id}`):** \* *Purpose:* A distraction-free page for taking an assessment using Shadcn `Radio Group` components for answers. Includes a submit button and an immediate pass/fail feedback `Alert`.  
* **Certificate View / Download Page (`/certificates/{uuid}`):** \* *Purpose:* A dedicated page displaying the generated certificate. Can be made publicly viewable so learners can share their achievements on LinkedIn or with employers.  
* **Profile Settings (`/profile`):** \* *Purpose:* Where users manage their name, email, connected SSO accounts, and view a history of their completed courses.

#### **C. Admin & Content Manager Pages (Internal / Secure)**

These pages utilize a separate "Admin Layout" (usually a persistent left-hand sidebar for navigation). Access is strictly controlled by Role-Based Access Control (RBAC) middleware.

* **Admin Dashboard (`/admin/dashboard`):** \* *Purpose:* High-level overview using Shadcn `Card` components to display metrics: Total Users, Total Enrollments, Course Completion rates.  
* **Course Management List (`/admin/courses`):** \* *Purpose:* A Shadcn `Data Table` listing all courses with columns for status (Draft, Published), author, and quick actions (Edit, Delete).  
* **Course Settings & Meta Editor (`/admin/courses/{id}/edit`):** \* *Purpose:* Form to edit the course title, description, cover image, and SEO metadata.  
* **Curriculum Builder (`/admin/courses/{id}/curriculum`):** \* *Purpose:* A highly interactive drag-and-drop interface. Admins can create Sections, add Lessons within those sections, and reorder them.  
* **Lesson Content Editor (`/admin/lessons/{id}/edit`):** \* *Purpose:* Where the actual content is uploaded or written. Includes file uploaders for S3 (videos/PDFs) or a rich text editor for text-based lessons.  
* **Quiz & Assessment Builder (`/admin/quizzes/{id}/edit`):** \* *Purpose:* Interface to add questions to a lesson, define the multiple-choice options, select the correct answer, and set the passing percentage.  
* **User Management (`/admin/users`):** \* *Purpose:* A Shadcn `Data Table` of all registered learners. Allows admins to search by email, view a user's progress, or manually revoke access if needed.  
* **Platform Settings (`/admin/settings`):** \* *Purpose:* Global configuration. Managing global SEO tags, updating the platform logo, or configuring API keys (if exposed to the Super Admin).

## **4\. UI/UX & Shadcn UI Component Mapping**

To maintain design consistency and speed up development, the team must prioritize official Shadcn UI components. Here is the mapping for your core LMS features:

* **Global Layout & Navigation:**  
  * `Navigation Menu`: For the top header (Catalog, Dashboard, Login).  
  * `Sheet`: For mobile off-canvas navigation and the Course Player's mobile curriculum sidebar.  
  * `Dropdown Menu`: For the user profile actions (Settings, Logout) in the header.  
* **Course Catalog & Dashboard:**  
  * `Card`: The primary container for displaying courses in the catalog (includes course image, title, and description).  
  * `Badge`: To display tags like "Beginner", "Intermediate", or "New".  
  * `Progress`: The linear progress bar showing course completion percentage on the dashboard.  
  * `Button`: Standardized primary and secondary actions ("Start Course", "Resume").  
* **Course Player:**  
  * `Accordion`: To display the Course Curriculum in the sidebar (Sections as the headers, Lessons as the collapsible content).  
  * `Checkbox`: Used visually in the curriculum accordion to denote completed lessons.  
  * `Separator`: To cleanly divide the video player from the text description below it.  
* **Quizzes:**  
  * `Radio Group`: For multiple-choice and true/false quiz questions.  
  * `Form` & `Input`: For any short-answer inputs (if added later) and handling validation states.  
  * `Alert`: To display passing or failing feedback immediately after quiz submission.  
* **Admin Dashboards:**  
  * `Table` (Data Table): For User Management and the list of Courses in the admin backend.  
  * `Tabs`: To switch between "Course Details", "Curriculum Builder", and "Settings" when editing a course.  
  * `Dialog` (Modal): For confirming destructive actions (e.g., "Are you sure you want to delete this lesson?").  
  * `Toast`: For global success/error notifications (e.g., "Course published successfully\!").

## **5\. Non-Functional Requirements**

* **Performance:** The application must operate as a Single Page Application (SPA). Page transitions should be instantaneous without full browser reloads (handled naturally by Inertia.js).  
* **Media Delivery:** Video streaming must not hit the Laravel application server directly. All videos must be served via a CDN or a third-party host (Vimeo/Mux) to ensure smooth playback for hundreds of concurrent free users.  
* **Security:** Standard Laravel protections (CSRF, XSS) apply. API routes handling progress tracking and quiz submissions must be strictly authenticated and rate-limited to prevent abuse/auto-completion scripts.  
* **Responsive Design:** 100% mobile-friendly. The course player must degrade gracefully to smaller screens, pushing the curriculum sidebar below the video or into a `Sheet` (drawer).
